JS javascript： 是前端编程的一门核心语言，使页面具有交互性 js用途： 1。可以读写html的内容 2.对事件进行处理 3.数据验证 4.可以操作cookie 5.模拟动画 6。服务端的应用（nodejs) JS的发展历史： ECMAScript: js是ECMAScript的实现 flash 1.1997年 Javascript1.1提交(EZMA) 2.1998年 2.0 3.1999年 3.0 4.2007年 4.0 5.2009年 5.0 6.2015年 6.0（es6），重大改变 7.2016年 7.0 js语法特点： 基于对象和事件驱动的松散型，解释型语言 1.基于对象： 面向对象开发 2.事件驱动： 可以处理事件 3.松散型,弱类型 可以使用一个关键字，声明各种数据类型的变量 松散型要区分 弱类型不用区分 4.解释型 由宿主环境（浏览器）解释执行 js的引入方式: 1.用 2. js注释： //注释本行 /* 和*/ 中间都被注释，注释很多行 js的输入工具： alert("内容"); //弹出框 prompt("请输入内容")； //输入框 注释快捷键：ctrl+？ confirm("提示的内容"); //提示框 var name = prompt("请输入姓名:") document.write("字符串")； //输入页面上 console.log("输出到控制台")； //输出到控制台 变量：存储数据的容器 var: 1.先声明，后赋值 var username; username = "小明（代名词）" 2.声明的同时赋值： var username = "小明"; 3.同时声明多个变量： var username = "小明",password=""; 特性： 1.变量声明提升 2.可以重复声明 1>重复声明时没有赋值，结果就是之前的值 var username = "张三"； var username; console.log(username);//张三 2>重复声明时赋值，变量的值会改变 var username = "张三"； var username = "李四"； console.log(username);//李四 let(es6): 特性： 1>不会提升变量声明 2>不能重复声明 const(es6)： 声明常量 特性： 1>必须在声明的同时赋值 2>变量的值不会改变 变量名的规范： 1.严格区分大小写 2.不能使用关键字或者保留字(var.let.const.in.delete.new.async) 3.开头必须是字母下划线、_、$，之后可以是数字.自贸区下划线、_、$ 4.驼峰命名法：首字母大写 5.变量名一定要有实际意义
